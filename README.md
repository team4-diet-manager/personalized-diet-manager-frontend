# Personalized Diet Manager — Frontend

[Personalized Diet Manager 백엔드](https://github.com/team4-diet-manager/personalized-diet-manager-backend)와 연동되는 식단·칼로리 관리 화면입니다. 온보딩 → 목표 선택 → 프로필 입력 → 대시보드(리포트) → 식단/체중 기록까지 단계별 흐름으로 구성된 SPA입니다.

## 기술 스택

- React 18, TypeScript
- Vite
- react-router-dom (라우팅)
- lucide-react (아이콘)
- 차트는 외부 라이브러리 없이 **SVG로 직접 구현**

## 주요 기능

- **단계별 온보딩**: 목표 선택 → 신체 정보 입력 → 권장 칼로리·탄단지 계산
- **대시보드**: 칼로리 링(권장 vs 섭취), 탄단지(섭취/권장) 막대, **최근 7일 칼로리 추이** 그래프
- **식단 기록**: 음식 **검색 콤보박스**(목표별 **신호등** 색상 표시), **최근 음식 1탭 추가** 칩, **식사 구분별 그룹핑·소계**, 등록/**수정·삭제**
- **체중 기록**: 날짜별 체중 입력(같은 날짜 갱신)과 **변화 그래프**
- **자동 날짜 전환**: 자정이 지나면 오늘 기준 화면으로 자동 갱신
- **새로고침 유지**: 프로필을 localStorage에 보관, 서버에 프로필이 없으면 온보딩으로 자동 복귀

## 실행 방법

```bash
npm install
npm run dev
```

- 개발 서버 주소: <http://localhost:5173>
- **백엔드(8080)가 먼저 실행되어 있어야 합니다.** `vite.config.ts`에서 `/api` 요청을 `http://localhost:8080`으로 프록시하므로 별도 환경 변수 설정 없이 동작합니다.

### 기타 스크립트

```bash
npm run build     # 타입 체크(tsc) + 프로덕션 빌드
npm run preview   # 빌드 결과 미리보기
npm run lint      # ESLint
```

## 화면 구성 (라우트)

| 경로 | 화면 | 설명 |
|------|------|------|
| `/` | 온보딩 | 서비스 소개 → 시작 |
| `/goal` | 목표 선택 | 다이어트 / 벌크업 / 체중 유지 |
| `/profile` | 프로필 입력·수정 | 성별·나이·키·몸무게·활동량 입력 후 저장 |
| `/dashboard` | 대시보드 | 칼로리 링, 탄단지(섭취/권장), 최근 7일 추이 |
| `/log` | 식단 기록 | 음식 검색·신호등, 최근 음식 칩, 식사별 그룹핑, 수정/삭제 |
| `/weight` | 체중 기록 | 체중 입력과 변화 그래프 |

상단 네비게이션(대시보드 / 식단 기록 / 체중 기록 / 프로필 수정)은 프로필이 저장된 뒤에 표시되며, 프로필이 없으면 목표 선택 단계로 유도됩니다.

### 동작 참고

- 리포트·기록은 모두 **"저장된 프로필" 기준**으로 계산됩니다. 프로필을 바꾸려면 상단 **프로필 수정**으로 진입해 저장하세요.
- 음식 **신호등 색상**은 저장된 프로필의 **목표 기준**으로 표시됩니다(목표를 바꾸면 자동 재분류).
- **백엔드를 재시작하면** in-memory DB가 초기화되어 저장된 프로필이 사라지는데, 이 경우 앱이 이를 감지해 자동으로 온보딩으로 되돌립니다.

## 백엔드 연동
